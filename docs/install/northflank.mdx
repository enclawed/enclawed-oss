Deploy Enclawed on Northflank with a one-click template and access it through the web Control UI.
This is the easiest "no terminal on the server" path: Northflank runs the Gateway for you.

## How to get started

1. Click [Deploy Enclawed](https://northflank.com/stacks/deploy-enclawed) to open the template.
2. Create an [account on Northflank](https://app.northflank.com/signup) if you don't already have one.
3. Click **Deploy Enclawed now**.
4. Set the required environment variable: `ENCLAWED_GATEWAY_TOKEN` (use a strong random value).
5. Click **Deploy stack** to build and run the Enclawed template.
6. Wait for the deployment to complete, then click **View resources**.
7. Open the Enclawed service.
8. Open the public Enclawed URL at `/enclawed` and connect using the configured shared secret. This template uses `ENCLAWED_GATEWAY_TOKEN` by default; if you replace it with password auth, use that password instead.

## What you get

- Hosted Enclawed Gateway + Control UI
- Persistent storage via Northflank Volume (`/data`) so `enclawed.json`,
  per-agent `auth-profiles.json`, channel/provider state, sessions, and
  workspace survive redeploys

## Connect a channel

Use the Control UI at `/enclawed` or run `enclawed onboard` via SSH for channel setup instructions:

- [Telegram](/channels/telegram) (fastest — just a bot token)
- [Discord](/channels/discord)
- [All channels](/channels)
